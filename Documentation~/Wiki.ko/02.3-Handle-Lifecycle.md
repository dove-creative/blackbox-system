## 1. 핸들 생명주기 개요

Blackbox는 스코프 닫기와 상호작용 병합을 `using` 흐름에 맞추기 위해 값 타입 핸들을 사용한다. `ScopeHandle`은 열린 스코프를 닫고, `ExertHandle`은 받는 쪽 스코프 병합을 시도한다.

두 핸들은 모두 `public readonly struct`이다. 값 타입은 복사될 수 있으므로, 복사본마다 `Dispose()`가 반복 실행되어도 실제 후처리는 한 번만 일어나야 한다. 이 중복 방지는 `HandleManager<T>`가 맡는다.

따라서 핸들의 생명주기는 세 단계로 볼 수 있다.

1. 기록 API가 핸들 id를 발급받아 핸들을 만든다.
2. 호출자가 `using` 또는 직접 `Dispose()`로 후처리 시점을 알린다.
3. `HandleManager<T>`가 살아 있는 id인지 확인한 뒤, 첫 dispose만 실제 로그 후처리로 통과시킨다.

## 2. 스코프 핸들

`ScopeHandle`은 `Scope(...)`가 만든 open 로그를 닫기 위한 핸들이다.

생성 시점에는 다음 정보를 가진다.

- `_id`: `HandleManager<ScopeHandle>`에서 발급한 중복 방지 id
- `_createdThreadId`: 핸들이 만들어진 스레드 id
- `_context`: close 요청을 전달할 `LogContext`
- `_scopeId`: open/close 로그를 연결할 스코프 id
- `_tagHandle`: 스코프 시작 로그에 관련 대상을 붙이기 위한 보조 값

`Dispose()`가 호출되면 먼저 id가 유효한지 확인한다. 이미 dispose된 id이거나 기본값이면 아무 일도 하지 않는다. 유효한 첫 dispose라면 출력 완료 여부와 문맥 존재 여부를 확인한 뒤, 생성 스레드와 dispose 스레드가 다른지 경고하고 `LogContext.CloseScope(...)`를 호출한다.

`With(...)`는 스코프 시작 로그에 태그 대상을 붙인다. 반환값은 같은 `ScopeHandle`이므로, 호출자는 태그 연결 뒤에도 동일한 스코프 생명주기를 `using`으로 유지할 수 있다.

## 3. 상호작용 핸들

`ExertHandle`은 `Exert(...)`가 만든 상호작용 로그를 받는 쪽 스코프와 병합하기 위한 핸들이다.

생성 시점에는 다음 정보를 가진다.

- `_id`: `HandleManager<ExertHandle>`에서 발급한 중복 방지 id
- `_createdThreadId`: 핸들이 만들어진 스레드 id
- `_targetContext`: 받는 쪽 상호작용 로그가 저장된 `LogContext`
- `_interactionId`: 양쪽 상호작용 로그를 연결하는 id

`Dispose()`가 호출되면 `ScopeHandle`과 같은 방식으로 id 중복을 먼저 막는다. 유효한 첫 dispose라면 출력 완료 여부와 문맥 존재 여부를 확인하고, `TryMergeScope(...)`로 받는 쪽 상호작용 로그 직후의 open 스코프를 찾는다.

바로 이어지는 open 스코프가 있으면 상호작용 로그는 무효화되고, 스코프 시작 로그가 상호작용 정보를 품는다. 조건에 맞는 스코프가 없으면 상호작용 로그는 한 줄 기록으로 남는다.

병합은 생성 시점에 고정된 받는 쪽 `LogContext` 안에서만 시도된다. 다른 스레드 문맥에서 나중에 생긴 스코프까지 임의로 합치지 않는다.

## 4. 핸들 id 관리

`HandleManager<T>`는 값 타입 핸들의 id를 발급하고, 살아 있는지 여부를 중앙에서 관리한다. 같은 핸들 값이 복사되어도 id는 같으므로, 첫 dispose만 성공하고 이후 dispose는 실패한다.

id는 슬롯과 세대를 함께 담는 값으로 만들어진다. 슬롯은 관리 배열의 위치이고, 세대는 예전에 해제된 슬롯을 다시 사용할 때 오래된 핸들이 새 슬롯 주인처럼 보이지 않도록 구분하는 값이다.

기본 구현은 lock-free 경로를 사용한다. `BLACKBOX_LOCK_HANDLES` 심볼이 있으면 명시적 잠금 기반 구현으로 바뀐다. 두 구현은 같은 외부 책임을 가진다.

- `GetHandleId()`: 새 핸들 id를 발급한다.
- `TryDisposeHandleId(...)`: 살아 있는 id를 dispose 상태로 바꾼다.
- `IsAlive(...)`: id가 아직 살아 있는지 확인한다.
- `WarnIfThreadMismatch(...)`: 생성 스레드와 dispose 스레드가 다를 때 경고한다.
- `Reset()`: 안전한 초기화 시점에 id 관리 상태를 비운다.

---

## 5. 구현 시 고려사항

핸들은 만든 스레드에서 dispose하는 것이 기본 흐름이다. 다른 스레드에서 dispose하면 기록 구조가 의도와 달라질 수 있으므로 경고를 남긴다.

출력이 끝난 뒤에는 `Infrastructure.IsPrinted`가 true가 된다. 이 상태에서 남은 핸들이 dispose되어도 close 로그나 병합 처리는 추가되지 않는다.

`BlackboxHandle.ForceReset()`은 registry와 runtime뿐 아니라 `HandleManager<T>` 상태도 초기화한다. 따라서 아직 dispose되지 않은 핸들이 남아 있는 실행 중에는 호출하지 않아야 한다.

reset 이후의 오래된 핸들은 더 이상 신뢰할 수 없다. 반복 실행이나 테스트에서 reset이 필요하다면, 이전 기록 흐름이 모두 끝난 안전한 시점에서만 호출해야 한다.
