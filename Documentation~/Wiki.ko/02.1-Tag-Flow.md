## 1. 태그 흐름 개요

Blackbox의 태그 흐름은 한 줄 로그나 스코프 시작 로그에 관련 객체를 붙여, 나중에 출력에서 함께 읽을 수 있게 만드는 구조이다.

상호작용 기록이 두 객체 사이의 호출 흐름을 양쪽에 남기는 구조라면, 태그는 원본 로그에 관련 대상을 표시하고 대상 객체 쪽에는 보조 태그 로그를 남기는 구조이다. 따라서 태그는 호출 구간을 만들지 않고, 이미 기록된 로그 한 줄에 관련 대상을 연결한다.

이 흐름의 중심은 `TagHandle`이다. `Write(...)`는 기록 직후 `TagHandle`을 반환하고, `Scope(...)`는 스코프 시작 로그에 태그를 붙일 수 있도록 `ScopeHandle` 안에 `TagHandle`을 보관한다.

## 2. 핵심 구성 요소

- `TagHandle`: 기록 직후 `.With(...)`를 받을 수 있는 값이다. 유효한 `LogContext`를 가지고 있으면 원본 로그에 대상 목록을 확정하고, 유효하지 않으면 대체 메시지만 반환한다.
- `TargetTypes`: 대상 쪽 태그 로그에 원본 정보를 어느 정도 표시할지 고르는 정책이다.
- `LogContext.ResolveWith(...)`: 원본 로그의 `Tags`를 채우고, 필요한 대상 객체에 태그 로그를 추가한다.
- `Blackbox.Tag(...)`: 대상 객체 쪽에 보조 태그 로그를 저장한다.
- `LogFormatter`: `%0`, `%1` 같은 치환 자리를 실제 대상 표시 문자열로 바꾸고, 메시지 안에서 쓰이지 않은 대상은 추가 태그 정보로 붙인다.

이 구성 때문에 태그 기록은 기록 시점과 출력 시점의 책임이 나뉜다. 기록 시점에는 원본 로그와 대상 목록을 구조적으로 보관하고, 출력 시점에는 `LogFormatter`가 사용자에게 보이는 문자열을 만든다.

## 3. `.With(...)` 실행 순서

1. `Write(...)` 또는 `Scope(...)`가 원본 로그를 저장한다.
2. 저장된 로그 위치와 대체 메시지를 가진 `TagHandle`이 만들어진다.
3. 호출자가 `.With(...)`에 관련 객체 목록을 넘긴다.
4. 마지막 인자가 `TargetTypes`이면 대상 표시 정책으로 사용하고, 대상 객체 목록에서는 제외한다.
5. null이 아닌 객체는 `BlackboxRegistry.GetBlackbox(...)`를 통해 `Blackbox`로 변환한다.
6. null 대상은 null 상태로 남겨, 호출자가 의도적으로 비어 있는 대상을 표시했음을 보존한다.
7. `LogContext.ResolveWith(...)`가 원본 로그의 `Tags`와 `TagTargetTypes`를 확정한다.
8. 대상이 null이 아니고 원본 객체와 다르면, 대상 객체의 `Blackbox.Tag(...)`가 보조 태그 로그를 남긴다.
9. `.With(...)`의 반환 문자열은 `LogContext.RenderMessage(...)`를 거쳐 최종 메시지 형태로 변환된다.

이 순서는 원본 로그가 먼저 존재해야 한다는 점이 중요하다. 태그는 새 기록을 시작하는 기능이 아니라, 바로 앞에서 만들어진 원본 로그에 관련 대상을 해석해 붙이는 후속 처리이다.

## 4. 대상과 표시 정책

메시지 안의 `%0`, `%1`은 `.With(...)`로 넘긴 대상 순서에 대응한다. 예를 들어 첫 번째 대상은 `%0`, 두 번째 대상은 `%1`에 들어간다. 메시지에서 쓰이지 않은 대상은 출력 끝에 추가 태그 정보로 붙는다.

대상이 없는 호출과 null 대상은 구분된다.

- `.With()`: 연결할 대상이 없는 호출이다.
- `.With(null)`: null 대상 하나를 명시적으로 연결한 호출이다.

대상 쪽 보조 태그 로그의 표시 범위는 `TargetTypes`가 정한다.

- `TargetTypes.None`: 최소 태그 표시만 남긴다.
- `TargetTypes.Name`: 원본 객체 이름과 id를 표시한다.
- `TargetTypes.InteractionId`: 태그 연결 id를 표시한다.
- `TargetTypes.Message`: 원본 메시지를 표시한다.
- `TargetTypes.Full`: 가능한 원본 정보를 모두 표시한다.
- `TargetTypes.BasedOnSettings`: 전역 설정의 기본 정책을 따른다.

이 정책은 원본 로그의 메시지 치환 방식이 아니라, 대상 객체 쪽에 추가되는 보조 태그 로그의 정보량을 조절한다.

## 5. 구현 시 고려사항

태그 대상은 객체 상태의 복사본이 아니다. 기록 시점에 대상 객체를 `Blackbox`로 해석하고, 출력 시점에는 해당 `Blackbox`의 표시 이름과 id를 사용한다.

대상 쪽 보조 태그 로그는 상호작용 출력 그래프를 확장하는 연결선이 아니다. `.With(...)`는 관련 대상을 함께 읽기 위한 보조 연결이고, `Exert(...)`처럼 호출 관계를 추적하는 흐름과는 역할이 다르다.

`TagHandle.FromMessage(...)`는 유효한 기록 문맥이 없을 때도 기존 메시지 흐름을 깨지 않기 위한 대체 경로이다. 이 경우 `.With(...)`를 호출해도 태그 해석은 일어나지 않고, 보관된 메시지만 반환한다.

전역 대상 표시 정책을 바꿔야 한다면 기록이 시작되기 전에 설정하는 것이 안전하다. 기록 이후에는 이미 남긴 태그 로그의 정책과 새 로그의 정책이 섞일 수 있다.
