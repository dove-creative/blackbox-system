# 1. 출력 파이프라인 개요

Blackbox의 출력 파이프라인은 기록된 `LogData`를 읽기 좋은 파일로 변환하는 흐름이다. 이 과정은 원본 로그를 새 의미로 바꾸기보다, 출력에 필요한 그래프 수집과 표시용 변환을 적용한다.

`BlackboxHandle.Export(...)`는 기준 객체의 `Blackbox`를 얻고, 현재까지 발급된 sequence를 기준점으로 잡은 뒤, 설정된 출력 형식에 맞는 출력기를 호출한다. 기준점 이후의 로그는 이번 출력 결과에 섞이지 않는다.

출력은 크게 세 단계로 나뉜다.

1. 기준 객체와 관련 객체를 모아 출력 그래프를 만든다.
2. 각 객체의 로그를 출력용 목록으로 변환한다.
3. 텍스트 또는 HTML 형식으로 파일을 쓴다.

---

# 2. 출력 그래프 수집

`BuildExportGraph(...)`는 출력에 포함할 `Blackbox` 목록을 `ExportNode` 목록으로 만든다. 첫 노드는 사용자가 출력한 기준 객체이며, 이후 노드는 상호작용 로그에서 발견한 관련 객체이다.

수집은 다음 규칙을 따른다.

- 방문한 `Blackbox`는 다시 방문하지 않는다.
- `recursionDepth`가 0 이상이면 해당 깊이까지만 따라간다.
- 일반 집중 출력은 `ExertedBy` 방향을 따라가, 기준 객체에 영향을 준 객체를 포함한다.
- 전체 출력은 `ExertedBy`에 더해 `ExertingTo` 방향도 따라가, 기준 객체가 영향을 준 객체도 포함한다.
- `.With(...)`로 만든 태그 대상은 관련 표시 정보일 뿐이므로 출력 그래프의 확장 경로로 사용하지 않는다.

각 `ExportNode`는 현재 객체, 부모 객체, 출력 깊이, 변환된 로그 목록을 가진다. HTML 출력기는 이 정보를 사용해 객체 단위 상자를 만들고, 객체 사이의 포함 관계를 표시한다.

# 3. 출력용 로그 변환

출력 그래프에 포함된 각 객체는 자기 `LogContext`의 로그를 모아 하나의 출력 목록으로 합친다.

변환 흐름은 다음 순서로 진행된다.

1. `GetLogsByContext(maxSequence)`가 기준 sequence까지의 스레드별 로그 목록을 가져온다.
2. `FlattenSteps(...)`가 내용 없는 open/close 쌍을 `ScopeType.Step` 로그처럼 보이도록 정리한다.
3. `ResolveScopeDepths(...)`가 open/close 흐름을 읽어 표시 깊이를 계산한다.
4. `Blackbox.MergeLogContext(...)`가 여러 스레드의 로그를 전역 sequence 기준으로 다시 정렬한다.

이 변환은 출력 전용이다. 원본 `LogData`가 가진 기록 의미를 바꾸지 않고, 사용자가 보기 좋은 깊이와 한 줄 표시 형태를 계산한다.

# 4. 텍스트/HTML 출력

`TxtExporter`는 각 로그를 `LogFormatter.RenderTextLine(...)`에 넘겨 한 줄 문자열로 만든다. 구조는 단순하지만, 스코프 깊이, 메서드 표시, 태그 치환, 상호작용 화살표는 `LogFormatter`의 공통 규칙을 따른다.

`HtmlExporter`는 같은 로그 데이터를 HTML 문서 구조로 배치한다. 각 로그 행에는 sequence, 스레드 정보, 시간, 메서드 표시, 메시지, 상호작용 연결이 들어간다. 상호작용 상대가 이번 출력에 포함되어 있으면 클릭 가능한 연결로 만들고, 포함되어 있지 않으면 비활성 표시로 남긴다.

HTML 출력에서는 `.With(...)`로 생긴 태그 연결 id를 메시지 본문 안의 대상 이름에 직접 붙이지 않고, `[id]` 형태의 주황색 태그 배지로 표시한다. 따라서 본문은 객체 이름을 읽기 좋게 유지하고, 연결 id와 이동 기능은 배지에서 담당한다.

`HtmlExporter.Templates`는 HTML 출력에 필요한 스타일과 동작을 제공한다. 접기/펼치기, 상호작용 행 이동, 강조 효과, 사용자 환경의 움직임 감소 설정 같은 화면 동작은 이 파일에 모여 있다.

메시지 자체의 의미 조립은 `LogFormatter`가 담당한다. HTML 출력기는 HTML 인코딩과 링크 표시를 맡고, 태그 치환이나 메서드 표시 규칙을 새로 해석하지 않는다.

# 5. 구현 시 고려사항

`LogDirectory`는 출력 파일을 쓸 위치이다. 출력기는 필요한 경우 디렉터리를 만들고, 파일을 쓴 뒤 경로를 로그로 남긴다.

출력은 기준 sequence를 가진 스냅샷이다. 출력이 시작된 뒤 새로 남은 로그는 같은 파일에 섞이지 않는다.

`CrashExport(...)`는 일반 출력 흐름을 사용하되, 파일 이름과 본문 앞쪽에 예외 정보를 더한다. 따라서 예외 출력도 같은 그래프 수집과 로그 변환 규칙을 공유한다.

`openLog`가 켜져 있으면 출력 뒤 파일 열기를 시도한다. 실패해도 출력 파일 자체는 유지되고, 실패 정보만 경고로 남긴다.

한 번 출력이 끝난 뒤에는 추가 기록과 중복 출력을 제한한다. 반복 실행이나 테스트처럼 새 기록 세트가 필요하면 안전한 시점에서 `BlackboxHandle.ForceReset()`을 호출해야 한다.
